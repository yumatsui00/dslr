# ロジスティック回帰について

 ロジスティック回帰は主に分類問題（特に２値分類）に使用され、出力が特定のクラス（ex: 0 or 1）に属する確率を予測する


## ロジスティック回帰の数式とシグモイド関数

ロジスティック回帰ではまず、線型結合を計算するが、その結果をそのまま出力するわけではない。

### 線型結合の計算
まず、入力変数とパラメータの線型結合を計算する。ここで、線型回帰の計算と同じ処理を行う。

### シグモイド関数で変換
シグモイド関数とは、S字型の曲線を描き、入力値を0から1の範囲に変換する非線形関数で、以下のように定義される。

![image](https://github.com/user-attachments/assets/f934862f-1c7f-41c2-af3a-85afc37df191)
![image](https://github.com/user-attachments/assets/b16268f3-4b6d-4d12-8f7d-418958409e53)

ロジスティック回帰では、シグモイド関数の出力を確率として解釈し、分類に使用する。具体的には、シグモイド関数を用いて、線型結合の結果```z = θ0 + θ1x1 + θ2x2 + ...```  を0から1の範囲にマッピングする。

例えば、ロジスティック回帰の予測が```z = θ0 + θ1x1 + θ2x2```のように計算されるとすると、これをシグモイド関数に通して確率に変換する。

****![image](https://github.com/user-attachments/assets/8a58f9b4-6600-48d5-a89c-6123b93c3f90)

ここで、　ｚによって変動する（正しくは個々のxの値によって変動）y が0.5以上であればクラス１，そうでなければクラス０と予測するように分類することができる。


### 目的関数
ロジスティック回帰では、対数尤度関数（たいすうゆうど）を最大化するようにパラメータを学習する。

![image](https://github.com/user-attachments/assets/2564938b-08b4-47e4-a931-9b2238c1d80f)

では、尤度関数とはなにか？
先ほど示したように、ロジスティック回帰では、各データポイントxiに対して、シグモイド関数を用いてクラス１に属する確率を計算する。
![image](https://github.com/user-attachments/assets/adfbadc7-0dd9-4fec-8b2d-ca1c22dd78b4)

尤度関数は、すべての観測データから得られるもっともらしさを示すもので、観測された各データポイントの発生確率の席として表される。

![image](https://github.com/user-attachments/assets/5fc33b05-cf9f-434c-a383-727b6e716833)

この式における、真ん中の辺はクラス１の確率を基準に取ったときの尤度関数で、右辺はクラス０を基準に取っている。（どっちも同じ）

この積は、データ数が多くなると、非常に小さな値となり、計算が難しくなるため、通常は対数を取って計算を簡単にする。これが対数尤度関数である。

![image](https://github.com/user-attachments/assets/074abe4c-31a0-4ebe-9a66-bf26b4514d96)

この対数尤度関数が最大化されるとき、データに最も適合するパラメータθを見つけることができ、それがロジスティック回帰モデルを最適化するための基本的な考え方となる。
